# Adoption Checklist

## Week 1

- [ ] Add `AGENTS.md` to repository root.
- [ ] Add PR template and evidence bundle template.
- [ ] Select risk-level definitions.
- [ ] Require evidence path in PRs.

## Weeks 2-4

- [ ] Add validation workflow.
- [ ] Create feature and bug brief examples.
- [ ] Train developers to convert conversations into briefs.
- [ ] Decide which tasks require human approval before agent edits.

## Month 2+

- [ ] Add project-specific test commands.
- [ ] Add hardware/firmware evidence capture if relevant.
- [ ] Integrate static analysis and security scanning.
- [ ] Track review time, rework, escaped defects, and evidence completeness.

## Metrics

- task success rate
- test pass rate
- review time
- rework count
- escaped defects
- evidence-bundle completeness
- prompt-to-merge time
- agent-caused regression count
