# Review Checklist

## Scope

- [ ] Patch matches the task brief
- [ ] No unrelated files changed
- [ ] Non-goals respected
- [ ] Public APIs changed only when approved

## Quality

- [ ] Code is understandable and maintainable
- [ ] Dependencies are justified
- [ ] Error handling is appropriate
- [ ] Observability/logging is appropriate

## Security and privacy

- [ ] No secrets committed
- [ ] Inputs validated where relevant
- [ ] Auth/authz unaffected or reviewed
- [ ] Personal/proprietary data not leaked into prompts, logs, or evidence

## Testing

- [ ] Unit tests pass
- [ ] Integration/regression tests pass if relevant
- [ ] CI passes
- [ ] Hardware/firmware evidence attached if relevant

## Evidence

- [ ] Evidence bundle exists
- [ ] Evidence matches risk level
- [ ] Residual risks documented
- [ ] Human approval recorded for L2+
